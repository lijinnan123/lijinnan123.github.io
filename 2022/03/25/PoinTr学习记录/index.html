名称 POINTATTN: YOU ONLY NEED ATTENTION FOR POINT CLOUD COMPLETION
时间 3.27
基本情况 这是改进transformer结构的文章（作者为浙大和南大的）
启迪 见下
重点 改结构中的seed generator有点意思，我稍微有点看懂了，其实这个和那个雪花transformer有点像，或者说如出一辙，这个模型结构好清晰：通过encoder进行特征提取之后得到特征码（几何形状码)，然后这个码通过seedG然后和原始点云重合，然后下采样一波，生成一个稀疏的点云骨干，然后该seed骨干和那个shape特征码一块儿进入point generator，生成最后的椅子。注意最后的那个generator是级联了两个！它还介绍说，他设计的这俩模块可以用到别的网络中，嘿嘿嘿（苍蝇搓手.jpg)它还说，这个用到了自注意力机制（残差网络形式的——从哪里体现出来的呢？)
疑问 就是不知道这玩意儿这么奇怪又简洁，是咋想出来的？最大的问题是那个用knn和不用knn都分别指的是什么呢？啥叫self-attention机制啊？FFN的作用是什么呢？为什么experiment单元的对比表格里面还有foldingnet啥的呢，这个是点云补全的模型吗？
  • 提出两个模块GDP和SFA，一个是几何细节感知块(GDP)，另一个是自特征增强块(SFA)。
  • PointAttN简单整洁，只包含最远点采样(FPS)、多层感知器(MLP)和注意层的下采样操作。
  • 消除了在特征学习中广泛使用的显式局部区域划分方法。
  • 通过利用交叉注意和自我注意机制，PointAttN可以精确地学习点之间的近程和长程结构相关性，这对解码阶段的细粒度信息恢复至关重要。
  • GDP和SFA借鉴了Transformer[17]的核心思想，即引入多头交叉注意和自我注意机制。
  • 但是，我们的方法没有采用Transformer的编解码结构。
  • 避免了knns的框架。
  • 我看着这玩意儿代码可简洁了，真的对学习这篇论文很有利啊，作者一定是一个精益求精强迫症。
  • 自我注意力机制和注意力机制有什么区别呢？
  • 特征提取模块：首先采用多层感知器(MLP)将无序点映射为特征向量。然后我们*交替叠加三个GDP块和三个SFA块，通过这些块可以在下采样过程中逐步嵌入局部和全局信息。设定三个GDP块的下采样比分别为4、2、2。这里设置三个SFA块的上采样比为1，即该模块中的SFA不改变feature的维数。堆叠完成后，采用另一个MLP作为上述FFN的角色***，并利用max pooling操作生成形状代码作为该模块的输出。
  • 引入GDP来感知点云的局部结构。同时，采用SFA增强模型在每次GDP运行后的全局信息感知。
  • 种子生成器：种子发生器在对输入的形状码进行解码的基础上，生成稀疏而完整的点云。首先通过MLP将形状编码解码为特征矩阵，再进行重塑操作，其中矩阵的每一行都可以认为是一个点的特征向量。通过另一个MLP扩展点特征的维度，将三个SFA块级联，增强对目标几何结构的特征感知能力。然后，通过重塑和MLP***对点特征进行分割得到稀疏点***，利用MLP将对应的特征向量转换为三维位置坐标。然后，通过串联将稀疏点与输入的部分点云合并。合并后的点云通过FPS下采样生成稀疏点云，作为下一个点发生器模块的种子点云。
名称 SnowflakeNet: Point Cloud Completion by Snowflake Point Deconvolution with Skip-Transformer

PF-NET的数据格式是：

——一串数字文件夹

——points_label文件夹(seg) && points文件夹(pts) && seg_img文件夹(png)

PCN数据格式是：

——test文件夹 && test_novel文件夹 && 一堆文件【train和valid相关】（.list文件和.lmdb文件和.imdb-lock文件）

shapenet的数据格式：

——train文件夹 && val文件夹

——complete文件夹 && partial文件夹
